# 08@LLMWiki 运行说明

## 目的
把 `03@Diary` 里的 `# 成功日记` 自动整理为 `08@LLMWiki` 下的日报、主题页、画像页、想法页和索引页。

## 手动运行

### 生成当天完整同步
```bash
node scripts/llmwiki.js sync --date YYYY-MM-DD
```

### 仅生成日报
```bash
node scripts/llmwiki.js daily --date YYYY-MM-DD
```

### 生成周报
```bash
node scripts/llmwiki.js weekly --date YYYY-MM-DD
```

### 生成月报
```bash
node scripts/llmwiki.js monthly --date YYYY-MM-DD
```

### 生成索引页
```bash
node scripts/llmwiki.js index --date YYYY-MM-DD
```

## 输入来源
- 读取 `03@Diary/YYYY/MM/YYYY-MM-DD.md`
- 读取 `07@WorkLog/YYYY/MM/YYYY-MM-DD.md`
- `03@Diary` 只分析 `# 成功日记` 下面的条目
- `07@WorkLog` 只读取当天工作日志
- 两者合并后进入 LLMWiki 系统

## 输出位置
- 日报：`08@LLMWiki/daily/YYYY-MM-DD.md`
- 周报：`08@LLMWiki/weekly/YYYY-MM-DD.md`
- 月报：`08@LLMWiki/monthly/YYYY-MM.md`
- 主题页：`08@LLMWiki/themes/*.md`
- 画像页：`08@LLMWiki/profile/个人画像.md`
- 想法页：`08@LLMWiki/ideas/YYYY-MM-DD-想法.md`
- 索引页：`08@LLMWiki/index/home.md`

## 长期知识规则
- `profile/个人画像.md` 是长期画像，不应被当天内容覆盖
- `themes/*.md` 是长期专题页，不应被当天内容覆盖
- 每次同步时应读取旧文档并做增量更新
- 建议保留 `history/` 快照，便于回看演化过程

## 自动化建议
### 第一阶段：定时任务
- 每天固定时间执行 `sync`
- 每周固定时间执行 `weekly`
- 每月固定时间执行 `monthly`

### 第二阶段：手动补跑
- 当天内容有改动时，可直接重新执行同一日期的命令覆盖输出

### 第三阶段：文件监听
- 后续如需更实时，可监听 `03@Diary` 目录变更，再延迟触发同步

## 故障处理
- 如果 LLM 调用失败，脚本会回退到本地 Markdown 兜底内容
- 如果没有生成结果，先检查：
  1. 当天日记是否存在
  2. `# 成功日记` 是否有条目
  3. OpenClaw / LLM 连接是否正常

## 备注
- 这套系统的目标不是写更多页，而是把碎片日记变成可复用的知识资产。
