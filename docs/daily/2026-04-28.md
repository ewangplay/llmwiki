## 问题或风险

- **发布链路对文件路径和命令配置较敏感**：同一课题在“完成”后还需要修复 Hexo 发布，说明自动发布环节仍有配置风险。
- **cron 对环境变量和 PATH 依赖较强**：如果没有单独的环境文件，脚本可能因找不到 Node 或相关变量而失败。
- **跨平台迁移存在遗漏风险**：Ubuntu crontab 是从 Windows 逻辑迁移过来的，worklog 任务最初缺失，说明迁移时需要逐项核对。
- **月报任务当前仍可能处于注释状态**：从输入里看，Ubuntu crontab 中月报尚未完全启用，后续需要确认。

## 可复用经验

- **课题完成后要顺手核验发布链路**，不要只看“已完成”。
- **遇到发布失败先区分内容问题和工程问题**，先查脚本和路径更高效。
- **跨平台定时任务迁移时，先补脚本再补调度**，这样更稳。
- **cron 配置要显式写环境变量**，不要依赖交互式 shell。
- **重复性任务适合整理成标准脚本**，后续日报、周报、worklog 都能复用同一套调度思路。
