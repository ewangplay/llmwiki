## 行动实验提炼

### 实验 1：用 Hermes 复刻一套最小可用工作流
- **假设**：如果 Hermes 更适合当前使用方式，那么用它搭一套最小工作流后，会比 OpenClaw 更省调教、更稳定。
- **来源**：[[03@Diary/2026/04/2026-04-14]] :: 15:33
- **触发条件**：下次需要快速搭建任务/日记/规则型工作流时。
- **验证方式**：
  1. 用 Hermes 重建一套任务与日记的最小工作流。
  2. 观察安装、配置、启动到稳定运行所需时间。
  3. 记录是否需要反复修补或额外调教。
- **预期价值**：判断 Hermes 是否真的更适合当前的工作流构建方式。

### 实验 2：把“执行日志是否清晰”作为工具选型标准
- **假设**：有完整执行日志的工具，更适合承载长期工作流，因为更容易排障和复盘。
- **来源**：[[03@Diary/2026/04/2026-04-14]] :: 15:33
- **触发条件**：以后比较任何 AI 工具或自动化方案时。
- **验证方式**：
  1. 记录每个工具是否提供步骤级日志。
  2. 比较出现问题时定位原因所需时间。
  3. 看是否能不用猜就知道流程卡在哪一步。
- **预期价值**：提高工作流的可维护性，减少“跑着跑着不知道为什么坏了”的情况。

### 实验 3：所有 AI 工具只按“是否服务工作流”来评估
- **假设**：如果把工具从“功能崇拜”拉回“服务工作流”，就能减少工具焦虑和无效试用。
- **来源**：[[03@Diary/2026/04/2026-04-14]] :: 21:48
- **触发条件**：每次想尝试新工具、切换工具、或比较工具时。
- **验证方式**：
  1. 每次试用前先写一句：这个工具要解决什么工作流问题。
  2. 试用后只回答一个问题：它是否真的让工作更顺。
  3. 如果答案是否定的，就停止继续投入。
- **预期价值**：防止被工具本身拖走注意力，保持目标清晰。

### 实验 4：把“新工具试用”限制为目标导向
- **假设**：如果试用新工具前就定义好目标和退出条件，就不会陷入反复折腾。
- **来源**：[[03@Diary/2026/04/2026-04-14]] :: 21:48
- **触发条件**：准备开始试用新的 AI 工具时。
- **验证方式**：
  1. 设定试用目标，比如“是否能稳定跑任务与日记工作流”。
  2. 设定试用时长和停止标准。
  3. 到点复盘是否达到目标，而不是继续凭感觉拖延。
- **预期价值**：让试用变成验证，而不是变成沉迷。

### 实验 5：保留“飞书规则直喂”作为快速启动路径
- **假设**：如果规则可以直接从飞书喂入，能更快建立可运行的工作流原型。
- **来源**：[[03@Diary/2026/04/2026-04-14]] :: 15:33
- **触发条件**：需要快速把规则变成可运行流程时。
- **验证方式**：
  1. 继续用飞书作为规则输入源。
  2. 观察从写规则到跑起来的耗时。
  3. 看后续是否方便迭代和修正。
- **预期价值**：减少搭建前置成本，尽快验证工作流可行性。

### 实验 6：把“工具只是手段”写成固定自检问题
- **假设**：如果每次试用工具前都问一句“这是在帮我，还是在消耗我”，就能减少工具迷失。
- **来源**：[[03@Diary/2026/04/2026-04-14]] :: 21:48
- **触发条件**：每次打开新工具、比较方案、或产生切换冲动时。
- **验证方式**：
  1. 在试用前手动写下这句自检问题。
  2. 试用后判断工具是增益还是负担。
  3. 观察这句话是否能显著降低冲动式尝试。
- **预期价值**：保持工具观清醒，把注意力留给真正重要的工作目标。

### 实验 7：优先选择“可稳定运行”的方案，而不是“看起来更强”的方案
- **假设**：对于日常工作流，稳定运行比表面功能更强更重要。
- **来源**：[[03@Diary/2026/04/2026-04-14]] :: 15:33
- **触发条件**：比较两个工具或两种方案时。
- **验证方式**：
  1. 对比连续运行 3 次的稳定性。
  2. 统计中断、报错、返工次数。
  3. 看哪个方案更少打断实际使用。
- **预期价值**：让选型更贴近长期效率，而不是短期新鲜感。

### 实验 8：限制工具焦虑的输入频率
- **假设**：如果降低对新工具信息的持续摄入，就能减少被工具洪流牵着走的感觉。
- **来源**：[[03@Diary/2026/04/2026-04-14]] :: 21:48
- **触发条件**：开始频繁关注新工具、新方案时。
- **验证方式**：
  1. 给自己设一个固定的工具观察窗口。
  2. 非窗口期不主动搜新工具。
  3. 观察焦虑感是否下降，工作流是否更稳定。
- **预期价值**：减少分心，把精力留给现有工作流的打磨。
