## 行动实验

### 1) LLMWiki 首页重构实验
- **假设**：如果按示意图重构首页，并优先恢复全站静态链接可用性，LLMWiki 的展示层会更清晰、更可浏览。
- **来源**：[diary] 13:33 关于站点构建、首页重设计、全站静态链接可用性检查的需求。
- **触发条件**：发现首页布局不符合预期、页面链接不可跳转或展示层显得凌乱时。
- **验证方式**：重构后逐页点击检查所有静态链接是否可用，并对照示意图确认首页结构是否更接近目标。
- **预期价值**：提升站点展示层的可用性和可读性，让 LLMWiki 从“能生成”升级为“能稳定浏览”。

### 2) 学习方法元技能实验
- **假设**：把“学习方法”当作可优化的元技能，而不是只盯知识点，能更有效地提升学习效率和兴趣。
- **来源**：[diary] 21:25 关于女儿学习瓶颈、元技能、思想实验、关卡式验证实践的分享。
- **触发条件**：孩子出现学习瓶颈、兴趣下降、效率不高或对具体知识学习感到枯燥时。
- **验证方式**：围绕同一学习内容尝试不同方法，记录哪种方法更能提升理解速度、兴趣和持续投入。
- **预期价值**：把学习从“死学”转为“会学”，增强可迁移的学习能力。

### 3) 亲子陪伴 + 身体锻炼实验
- **假设**：通过固定的亲子运动和学习交流，能同时改善关系、身体状态与学习信心。
- **来源**：[diary] 21:30 关于周末陪女儿打羽毛球、加强心得交流、增强体质与学习信心的安排。
- **触发条件**：发现孩子陪伴不足、身体状态较差、学习兴趣下降时。
- **验证方式**：周末持续执行“1小时羽毛球 + 学习交流”，观察孩子的身体状态、交流意愿和学习情绪变化。
- **预期价值**：用稳定陪伴替代零散说教，增强亲子关系并提升孩子的身体与学习状态。

### 4) 安全感驱动学习实验
- **假设**：如果先提升孩子的安全感和被理解感，再谈学习管理，学习态度更容易改善。
- **来源**：[diary] 21:35 关于青春期疏远、父母之爱、安全感与学习态度之间关系的反思。
- **触发条件**：孩子出现抵触、失去兴趣、与父母交流变少、学习管理松动时。
- **验证方式**：在日常交流中减少评判，增加倾听和理解，观察孩子是否更愿意沟通学习困难与情绪状态。
- **预期价值**：减少“只看表象”的误判，避免把学习问题简单归咎于不努力。

### 5) 阶段化提交实验
- **假设**：把个人知识系统按“工作日志同步 / LLMWiki 同步 / 站点源生成”分阶段提交，能更清楚地固定每一步产物，便于回溯和排错。
- **来源**：[diary] 23:23 对个人知识系统阶段化提交的优化方向。
- **触发条件**：每次完成一个关键同步阶段或生成阶段产物时。
- **验证方式**：连续一段时间按阶段提交，检查是否更容易定位问题、回滚变更、理解系统演进过程。
- **预期价值**：让系统开发过程更可控，也更利于长期维护。

### 6) 多 Agent 协作实验
- **假设**：把统筹、写作、深度讨论拆分给不同 Agent，并设置统一调度机制，能提升个人知识系统的协作效率。
- **来源**：[diary] 23:26 关于个人知识系统 1.0 运行、多 Agent 协作、统一调度代理的设想。
- **触发条件**：当前单 Agent 在统筹、写作、深度讨论上出现能力瓶颈时。
- **验证方式**：先用最小角色分工试运行，再观察不同 Agent 在任务类型上的表现差异和协作效率。
- **预期价值**：让系统更接近“分工明确的工作流”，减少单点能力不足带来的限制。

### 7) 统一协调 Agent 实验
- **假设**：如果引入一个统一协调 Agent 负责接收指令并分派任务，能减少用户手动指定每一步由哪个 Agent 完成的心智负担。
- **来源**：[diary] 23:29 关于个人知识系统 2.0 中“统筹 Agent 自动分派任务”的构想。
- **触发条件**：多 Agent 机制基本跑通后，开始设计 2.0 系统时。
- **验证方式**：让协调 Agent 接收添加日记、添加任务、设计模块、写作、深度讨论等指令，观察其分派是否合理、执行是否顺畅。
- **预期价值**：把个人知识系统从“多 Agent 并列”推进到“协同调度”，形成真正的自动分工模式。
